### 2.1 第一章：Cortex 是什么

- **核心定义**：LLM 驱动的个人工具链
- **核心隐喻**：工具链（从 v1.1 大脑/神经系统变更）
- 三大工具链特质（可替换/可验证/职责清晰；可审计；用户为最终裁决者）

### 2.2 第二章：七条不可变原则

| 原则 | 核心内容 |
|------|---------|
| 原则一 | 确认在用户手里。L2/L3 不可逆操作必须经用户确认 |
| 原则二 | 规划与执行分离。MetaAgent 只规划不执行，Agent 只执行不规划 |
| 原则三 | 安全边界在 Toolkit 调用层。Agent 以身份调用，不持有权限定义 |
| 原则四 | 谁调用谁负责。Agent 对其工具调用后果承担全部责任 |
| 原则五 | 所有可观测事件走 PipelineObserver 统一管道。SafeErrorReporter 定义 fatal/degraded/silent 三档 |
| 原则六 | 用户是最终裁决者。多 Agent 并行产出须先经圆桌协商收束再呈用户裁决 |
| 原则七 | 系统自我修改受宪法约束。Cortex 对自己的代码和文档的任何修改必须遵守七项子约束 |

**子章节**：

### 2.3 第三章：系统架构

**主干结构**：
```
Cortex
├── Engine（容器）
│   ├── MetaAgent（规划中枢）
│   ├── Agent池（11 Agent）
│   ├── TaskBoard（任务板，并发控制）
│   ├── ConfirmGate（确认门）
│   ├── PipelineObserver（可观测管道 + SafeErrorReporter）
│   ├── ConsistencyLayer（P1-六层防御——Facade）
│   │   ├── InitVerifier（启动校验）
│   │   └── SchemaEnforcer（结构校验）
│   ├── MemoryStore（运行时记忆，30天窗口，委托模式 Facade）
│   │   └── 内部委托组件（memory/ 子目录，8 组件族）
│   └── Scheduler（Agent 调度，拓扑排序→逐层并行）
├── LLM 适配层
│   └── LlmAdapter（API 适配、缓存、重试、流式、指纹匹配）
├── 基础设施
│   ├── Toolkit（工具目录与权限校验）
│   ├── FileLockManager（文件级锁）
│   ├── CLIAdapter（CLI 平台桥接）
│   ├── Core-2 预留：TrustModel / Sentinel
│   └── SkillRegistry（技能注册表）
├── 管家（独立进程）
│   └── Core-2 预留：消息源插件 / 周期性汇总简报
└── 治理层
    ├── 宪法（本文档）
    ├── 治理层设计
    ├── DocGovernAgent
    ├── 阶段门禁检查表

```
