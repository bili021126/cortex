## 一、核心变更对文档的总体影响

宪法 v2.0 以工具链隐喻替代大脑隐喻。以下 v1.1 核心概念全部废止：
- 19 功能柱 → 6 Agent 类型
- 三取向（管家/搭档/监理）→ Agent 类型化
- 事件总线/脊髓 → PipelineObserver
- 脑干/神经节点/小脑 → 消解或分散
- 三省六部 → 治理层（DocGovernAgent + 常设委员会 + 临时委员会）
- 交融机制（staining/lifecycle/projection）→ 开会
- CommitteeManager 代码 → 会议模式
- 监理封驳权 → 分散
- 民主集中制 → Expert Committee（用户裁决）

## 三、Meso-Lite 代码层面的影响

Meso-Lite 代码（10 包 monorepo）在 Core 阶段将**全面重构**，而非增量修改。保留的只有：
- 测试框架（mock 基础设施、E2E 管线）
- ConfirmGate 核心逻辑（确认门不变）
- CoroutineRunner 执行循环
- 工具调用底层适配

废弃的代码：
- 19 柱 PillarId 类型枚举 → 缩减为 6 AgentType
- 3 个 Pillar 实现类 → 6 Agent 配置
- InMemoryTransport + EventBus → PipelineObserver
- CommitteeManager → 删除
- 交融支撑组件（staining/lifecycle/projection）→ 删除
- CortextEngine 771行上帝类 → 拆分
- 3 个 ToolInvoker 子类 → 删除
