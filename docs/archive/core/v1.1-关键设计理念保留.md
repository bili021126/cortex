## 一、HCA/CSA 注意力架构（宪法 §9.1）

> 原始：Meta-Agent 用 HCA 做全局规划（略读——广而浅），功能柱用 CSA 做局部执行（精读——窄而深）

**核心理念**：规划与执行的上下文策略本质不同——全局需要广度牺牲深度，局部需要深度牺牲广度。一个 LLM 调用无法同时兼顾两者。

**在当前架构中的映射**：
- [has/cas 设计模式](file:///d:/cortex/docs/core/Core-1-第四轮-记忆系统设计反思与工程教训.md)：`has()` = 轻量存在断言（Meta-Agent 广而浅），`cas()` = 原子状态博弈（Agent 窄而深）
- Meta-Agent 注入全局压缩信息做方向性决策，Agent 聚焦当前任务节点细节
- **轻判断 + 重校验 pattern**：HCA/CSA 和 has/cas 从资源约束独立收敛到同一设计模式

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L725-L734

## 二、遗忘四态模型（宪法 §7.3）

> 原始：Active → Archived → Frozen → Obliterated，CAS-enforced 不可逆转

**核心理念**：
1. 记忆不仅有权重变化，还有清晰的生命周期阶段
2. 遗忘不是系统的缺陷，而是记忆系统完整性的最后一块拼图
3. 流转必须有原子性保障——CAS 状态机，不可回退
4. 解决了三个根本问题：无限积累的存储压力、过时经验污染新决策、删除后衍生记忆残留

**在当前架构中的映射**：
- MemoryState 四态枚举已在 [memory-store.ts](file:///d:/cortex/packages/engine/src/memory-store.ts) 定义
- CAS 状态机 + 审计 + 去副作用 read + batchTouch 列入 Core-1 第四轮优化
- 冻结态→湮灭态需监理审慎背书 + 用户显式确认（不可逆操作原则）
- 检索时一致性保障：所有检索路径经过同一个状态过滤模块

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L620-L627

## 三、单一基础关联网络 + 检索时动态投影（宪法 §7.2）

> 原始：写入时只描述事实本身，不预判哪个取向会用到。检索时按取向维度动态投影。

**核心理念**：
1. Write once, project at read time——写入不预判消费者
2. 三种投影规则预定义（代码维度/部署维度/生活维度），不靠 LLM 临场判断
3. 关联深度硬限制（Meta-Agent ≤3 跳，Agent ≤1 跳，管家 ≤1 跳跨源），防止索引爆炸
4. 关联定期裁剪：N 天未被访问的关联降权归档
5. 交叉参考受控：跨取向访问按关联链长度和历史相关性自动降权/呈现

**在当前架构中的映射**：
- LinkType 七枚举已在 [memory-store.ts](file:///d:/cortex/packages/engine/src/memory-store.ts) 定义
- BFS 检索 + 投影规则列入 Core-2 计划
- 关键词旁路已实现（当前主路径），BFS 为关联检索辅助路径
- 管家/搭档/监理三取向投影规则的配置化——在三份 `projection-rules.json` 中定义

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L563-L590

## 四、自迭代策略 + ε-贪心探索（宪法 §7.3）

> 原始：每次任务后追加策略评估——检索是否因缺少关键记忆走偏、是否因无关记忆太多干扰。评估结论经用户确认后覆盖元记忆区。ε-贪心探索防止回音室效应。

**核心理念**：
1. 自迭代策略是认知闭环的核心驱动——不是"写死的检索规则"，而是从历史经验中自我校准
2. 策略变更带版本号和审计日志，可追溯
3. ε-贪心探索（出厂基线 5%）：以 ε 概率随机选择非最高权重策略，打破早期偶然成功策略的路径锁定
4. 探索结果不入常规成功率统计，进独立多样性收益评估池
5. 衰减回退：高频使用被写为默认的参数，衰减期内不再触发则自动回退保守基线
6. ε 值本身也受自迭代调节——探索池长期无洞察则衰减，创造性参与率下降则调高

**在当前架构中的映射**：
- Core-3a（自迭代能力上位）：LoopAgent + retrieval_feedback 数据积累
- 冷启动观察期设计参考：新架构仍有冷启动问题，需要数据驱动的策略演化
- 策略评估→确认→覆盖的闭环在 v2.0 中简化为 `retrieval_feedback` 表结构

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L608-L617

## 五、隐私自限性（宪法 §2.6）

> 原始：Cortex 不为用户画像。不主动分析用户的生理、心理或生活行为模式，不将碎片数据聚合为健康画像或情绪判断。

**核心理念**：
1. 系统不主动聚合碎片数据为长期行为趋势
2. 医学信息处理的硬性边界：仅限同份报告内指标罗列 + 通用参考信息引用 + 硬编码关键词列表触发
3. 未经用户显式提供的数据，不做任何健康相关推断
4. 此原则是不可变内核，不随信任等级提升而放松

**在当前架构中的映射**：
- 管家取向的信息处理边界——呈现但不推断
- 季度报告中不聚合为推断语句（"本月 12 次凌晨会话"而非"入睡时间延后"）
- 私密记忆的物理隔离（管家取向专属）

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L126-L138

## 六、确认不可抵赖性 + 疲劳保护（宪法 §2.4）

> 原始：确认不能仅依赖用户文本输入或点击行为。须生成含操作内容哈希+用户身份凭证+确认时间戳的强审计记录。哨兵检测确认洪峰时触发注意力资源冲击告警。

**核心理念**：
1. 数字签名原则：操作内容哈希 + 用户 ID + 时间戳 + HMAC 签名
2. 确认疲劳保护：哨兵检测短时间窗口内确认请求数超阈值 → 全局广播降级指令
3. 管家注意力守门员：临时代理低风险批量确认，只把真正需要用户判断的不可逆操作呈现
4. "守护注意力与守护安全是同一件事"

**在当前架构中的映射**：
- ConfirmGate 已在 [engine/src/confirm-gate.ts](file:///d:/cortex/packages/engine/src/confirm-gate.ts) 实现，含 HMAC 签名
- 疲劳检测（连点 ≥5 次或延迟 <500ms）在 TrustModel 中设计
- 管家注意力守门员——当前仍在概念阶段

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L113-L115, L499-L500

## 七、六层提示词架构（宪法 §9.4）

> 原始：每次模型调用遵循同一套基于中文思维逻辑的认知框架——情境先行、关系本位、尚简留白。

**核心理念**：
1. 情境先行：先铺完整情境图，抑制情境缺失导致的幻觉
2. 关系本位：用关系准则取代行为禁令列表（"事关安全宁可多问"而非"禁止做X"）
3. 尚简留白：关键处点明，其余留给模型自主判断
4. 六层框架：当前情境 → 身份位置 → 分寸拿捏 → 任务范围 → 可用信息 → 输出规范
5. 输出前自我校验：哪一句必不可少？删了不影响决策的立刻删除

**在当前架构中的映射**：
- [meta-agent.ts](file:///d:/cortex/packages/engine/src/meta-agent.ts) 中的 REPLAN_SYSTEM 提示词
- Agent systemPrompt 模板中的身份定位和工具契约
- 六层框架可作为所有 LLM 调用的提示词结构标准

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L760-L789

## 八、信息熵分级控制（宪法 §9.2）

> 原始：高熵精简（结构化字段，极限压缩）→ 中熵标准（要点式陈述，每句有推理支撑）→ 低熵详尽（原始信息附前，提炼附后）

**核心理念**：
1. 系统内部流转禁止生成自然语言冗余——非面向用户的信息必须结构化
2. 面向用户的信息密度必须高于闲聊基线
3. 警惕"信息繁重堆砌"——未经提炼的原始细节堆叠同样是上下文浪费
4. 高熵 → PipelineObserver 事件、状态摘要、哨兵告警
5. 中熵 → Meta-Agent 规划上下文、Agent 任务描述
6. 低熵 → 用户对话、代码审查报告、事故复盘

**在当前架构中的映射**：
- PipelineObserver 事件格式已遵循高熵结构化设计
- Meta-Agent 规划指令使用中熵要点式陈述
- 上下文管理策略——Core-1 中 30 天热数据窗口已实现空间约束
- 信息熵约束可作为提示词工程中的输出格式标准

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L738-L747

## 九、五种闭环架构（宪法 §3）

> 原始：信号闭环 → 执行闭环 → 认知闭环 → 控制闭环 → 价值闭环，自底向上嵌套

**核心理念**：
1. 信号闭环（底层）：确保信息不丢失——每个关键信号都有明确归宿
2. 执行闭环（主体）：意图→规划→执行→验证→交付→反馈的完整流转
3. 认知闭环（进化）：经验→知识→策略优化→执行改善
4. 控制闭环（边界）：闭环内部自主推进，关键决策点向用户开门
5. 价值闭环（归旨）：所有活动最终回到用户——管家呈现成果、收集反馈

**在当前架构中的映射**：
- 信号闭环 → PipelineObserver 优先级回调 + node.* 事件全覆盖
- 执行闭环 → Scheduler 拓扑排序 + MetaAgent 规划 + Agent 执行 + 重规划
- 认知闭环 → Core-3a 自迭代（LoopAgent + retrieval_feedback）
- 控制闭环 → ConfirmGate + Committee 三级收束 + 用户裁决
- 价值闭环 → 管家交互通道 + 用户反馈闭环

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L154-L221

## 十、搭档反驯化机制（宪法 §10.1）

> 原始：搭档的质疑本能不可关闭。信任提升可放松确认门槛，但绝不降低质疑强度。

**核心理念**：
1. 四个递进层次：主动进击 → 数据打脸 → 冷嘲热讽 → 原则性沉默
2. 信任与质疑分离：L4 深度信任的搭档仍然会说"这样做可能有并发风险"
3. 锐度调节：从"直接中性"到"诤友模式"，但不可彻底关闭
4. 自动降锐：检测到用户连续收到负面反馈且响应率骤降时自动降低锐度
5. "用户可以决定不听，但不能让镜子停止反射"

**在当前架构中的映射**：
- ReviewAgent 的审查模式继承质疑本能——读+审计工具，不可关闭
- 锐度调节映射为 systemPrompt 中的语气调整参数
- 自动降锐映射为 TrustModel + 疲劳检测机制

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L794-L830

2. 每个缺陷都有明确的缓解策略和残余风险声明
3. 作为宪法的诚实补充——告诉读者"这里会出问题，我们能做的是压缩到可控范围"

## 十二、Committee 三级收束（宪法 §4.1.3）

> 原始：事实最高 → 基线优先 → 分歧明确交付

**核心理念**：
1. 事实最高：能引用经验记忆里具体事实的一方胜出
2. 基线优先：激进方案 vs 保守方案僵持，按出厂安全基线裁决
3. 分歧明确交付：双方都有事实但结论不同，不强行统一，写明分歧点呈用户确认
4. 事实引用的记忆态声明：归档态权重降半，冻结态不得引用为论据

**在当前架构中的映射**：
- Core-1 已部分实现——needsMultiPerspective 触发 + MetaAgent 裁决
- 事实最高/基线优先在 Core-1b Committee 三级收束中完整实现
- 记忆态声明在 MemoryStore CAS 状态机中实现

**参考**：`Cortex 概念顶层设计 v1.1-已废弃.md` L294-L303

## 十三、技能体系设计（超前设计）

> ⚠️ 超前设计。原型已验证（verification-templates-soft.json），工程落地前提不成立。
> 原始来源：v1.1 §4.2 小脑 + §7.3 自迭代 + §10.2 认知主权 + §11 半开放半封闭 + 小尾巴技能设计。

### 核心原则

**技能的本质**：约束 LLM 不确定性的确定性工具。技能不是给 Agent 的建议——是用来抑制 LLM 推理不稳定的硬约束。

**三环节归确定性，LLM 不进裁决链**：
- **匹配**：`tags ∩ projectId ∩ dependencies` 硬校验。不用 MetaAgent 选，不用 LLM 匹配
- **生命周期**：计数器驱动——`adoptCount≥N→晋升`，`rejectCount≥N→废弃`，`N天未触发→降级`。不用 Agent 审批
- **沉淀**：LoopAgent 提议 + 用户确认。唯一 LLM 入口在发现端，最终决策不在 LLM

**DocGovernAgent 重新定位**：不是技能裁判，是计数器的解释者。adoptCount 到了阈值，它写审计摘要；rejectCount 到了阈值，它归因分析。它不投票。

**一技能一节点，编排在 TaskBoard**：技能是一个 TaskNode，不是一个 Agent 的内部流水线。MetaAgent 拆任务时就完成了技能匹配和编排。Agent 是执行者，不是编排者。

**技能 = MemoryStore 里的记忆子集**：不是独立文件系统。`MemoryType.Skill` 是 MemoryStore 中被 LoopAgent 额外标记了"建议复现"的经验片段。归属追溯到对话记忆。

**用户不写技能**：一句话种子 → 系统自动补全为完整技能。小尾巴的精确工具调用格式不适用 Cortex——Cortex 的技能是中文情境提示词，不是工具调用脚本。

**原子化**：入口 + 出口 + 验证。不可拆分。原子是对意图而言的，不是对工具调用而言的。

**原型**：verification-templates-soft.json——7 个中文提示词式技能模板，5 轮运行验证通过。SkillExecutor / SkillRegistry / 自动沉淀 / 生命周期计数器——均为超前设计。
