# 预备修宪清单

**性质**：Meso 反思阶段识别的潜在违宪场景。已触发项在宪法 v2.4 中部分落地，未触发项保留至对应阶段启动时再启动正式修宪讨论。

**管理规则**：每次阶段跃迁的监理合规审查必须检查本清单——是否有任何项已被事实触发。

**版本**：v1.2（2026-05-04 宪法 v2.4 对账后刷新）
**创建日期**：2026-05-04
**关联**：Cortex 概念顶层设计 v2.4（当前生效版本）


## 第一类：宪法条款的工程可行性边界

### P-1：Committee 规模上限"5 个"的工程语义

- **宪法条款**：4.1.3 "规模硬性上限：五个功能柱专指参与专业辩论的功能柱"
- **潜在违宪场景**：19 柱全部上线后，同一任务节点可能同时涉及安全、性能、可维护性、UX、代码生成、部署六个视角。5 个上限是否包含 Meta-Agent？如果 Meta-Agent 不占席位但需协调 6 个功能柱，宪法"5 个"是硬性上限还是建议上限？
- **触发条件**：Core-2a 阶段 19 柱全量上线后，Committee 组建时出现超过 5 个专业视角同时请求的情况
- **预估阶段**：Core-2a

### P-2：脑干四类关键控制事件的穷举性

- **宪法条款**：4.3 "仅此四类，硬编码，不可扩展"
- **潜在违宪场景**：实施中可能出现宪法未预见的维生级需求（如 LanceDB 索引损坏导致记忆系统不可用、需要紧急重建），此类事件是否属于"关键控制事件"？不属于则脑干无法兜底，属于则需修宪扩展四类清单
- **触发条件**：Meso-Full 阶段完整脑干上线后，出现需要强制投递但不在四类清单内的事件
- **预估阶段**：Full-1a

### P-3：私密记忆"物理隔离"在不同运行时的最低工程标准

- **宪法条款**：4.5 "私密记忆的物理访问边界"
- **潜在违宪场景**：在单进程协程模型中，物理隔离实际上只是 `is_private` 字段 + 染色校验——没有独立进程、独立内存空间、独立文件。如果攻击者控制了 ToolGateway 代码路径，私密记忆的"物理隔离"实质上不存在
- **触发条件**：安全审计或渗透测试发现 ToolGateway 染色校验可被绕过
- **预估阶段**：Meso-Core

### P-4：工具调用层"唯一交互通道"的工程强制力

- **宪法条款**：4.5 "功能柱通过工具调用层获取外部数据、执行系统操作，但不得绕过此层"
- **潜在违宪场景**：TypeScript/JavaScript 运行时无法在编译期强制"禁止直接 import fs"。如果功能柱代码中直接写了 `import fs from 'fs'`，系统在运行时无法拦截。宪法的"不得绕过"在工程上能达到什么级别的强制力？当前只能靠 lint 规则和代码审查
- **触发条件**：代码审查发现功能柱绕过 ToolGateway 直接访问原生模块，或 lint 规则被禁用
- **预估阶段**：Meso-Lite 实施初期


## 第二类：宪法条款之间的内生张力

### P-5：原则三（事件不带指令）vs 脊髓反射（一级响应）

- **宪法条款**：2.3 "膝跳反射级协作直接在脊髓层完成" vs 原则三"横向事件不带指令"
- **潜在违宪场景**：脊髓反射中的"预注册反射规则"本质上是一条预存指令——"当 X 事件发生，执行 Y 操作"。反射规则的注册者、修改者、审计者的权限体系在宪法中未定义，可能成为原则三的隐蔽后门
- **触发条件**：Core-2c 阶段脊髓反射上线后，需要定义反射规则的注册/修改/审计权限
- **预估阶段**：Core-2c

### P-6：质疑本能不可关闭 vs 锐度调节的"自动临时降锐"

- **宪法条款**：原则五（搭配质疑本能不可关闭）vs 10.1 "搭档在检测到以下信号时自动临时降低锐度"
- **潜在违宪场景**：自动降锐期间，搭档是否仍在质疑只是表达方式变温和了？还是质疑本能也被临时压制了？"质疑本能"和"锐度"的边界需要更精确的定义
- **触发条件**：搭档自动降锐触发后，用户向搭档提出了需要质疑的技术决策，搭档的回应方式引发对"质疑本能是否被关闭"的疑问
- **预估阶段**：Core-2a


## 第三类：跨宪法条款的接口空白

### P-7：神经节点 vs Expert Committee 的冲突升级路径

- **宪法条款**：5.3（神经节点，执行层冲突）vs 4.1.3（Committee，设计层分歧）
- **潜在违宪场景**：执行层冲突的根因是设计层分歧时，神经节点局部协调后根因仍在，会反复触发。宪法目前没有定义"神经节点 → Committee"的升级通道
- **触发条件**：同一目标资源在滑动窗口内神经节点反复激活超过阈值，且根因分析指向设计层分歧
- **预估阶段**：Full-1b

### P-8：交融场景下记忆投影规则的冲突裁决

- **宪法条款**：6.3（交融场景）vs 7.2（记忆投影规则）vs 7.2（交叉参考受控）
- **潜在违宪场景**：管家 Filter 柱被搭档借调，检索走搭档投影规则。沿技术关联链触碰到管家维度的生活信息——这是合法的能力借用，还是需要权限校验的跨维度访问？"能力借用"和"交叉洞察"的边界不清晰
- **触发条件**：交融借调场景下，MemoryAccessor 检索路径跨越了主取向和辅助取向两个维度的记忆，且其中涉及私密信息或敏感模式
- **预估阶段**：Core-2a


## 第四类：Meso-Lite 实施阶段暴露的结构性漏洞

### P-9：阶段交付物自洽性验证机制缺失

- **宪法条款**：议题五 Section 4.9（退出标准）——宪法定义了"什么算达标"，未定义"达标声明的依据可否信"
- **潜在违宪场景**：阶段 README 宣告"6 大交付 100% 完成""退出标准 5/6 达标""测试 151 全绿"——这些声明本身没有跨文档交叉验证。Meso-Lite 实践证实：README 日期两次改错且无人察觉，预备修宪清单 P-4 判定"已过站未触发"推导链路无追溯。如果阶段关卡不设自洽性审查，退出声明可能系统性地偏离实现事实
- **触发条件**：已在 Meso-Lite 反思阶段触发——17 条文档错误在反思中被集中发现
- **预估阶段**：已触发，需在下一阶段设计阶段关卡自洽性校验机制

### P-10：单进程假设对架构正确性的遮蔽

- **宪法条款**：原则三（横向事件不带指令）——事件总线支持并发订阅者，但单进程协程模型下的 `Promise.allSettled` 掩盖了事件驱动架构的事务图景
- **潜在违宪场景**：Memory 写后读的事件链（写入 → 发布 MEMORY_WRITTEN → 订阅者查询）在单进程下时序稳定，但在并发场景下可能出现"订阅者读到更新前状态"或"两个订阅者竞态"。ToolGateway 静态注册（init 时逐个 register）掩盖了"动态注册无生命周期管理"的问题。关键词分类器（优先级链无歧义处理）在单取向输入下正确，多取向意图无回退路径
- **触发条件**：已在 Meso-Lite 反思阶段暴露——Memory 事务语义、ToolGateway 动态性、关键词歧义全部被单进程原型遮蔽
- **预估阶段**：已触发，Core 阶段引入并发/动态工具/多用户时必须直面

### P-11：常设委员会在架构中的空缺

- **宪法条款**：4.1.3（Committee 协作通道）——宪法定义了 Meta-Agent 主持的临时委员会（事件驱动、短生命周期、按议题召集），但未定义常设委员会
- **潜在违宪场景**：临时委员会处理"这个高风险操作要不要做"，但不处理"跨阶段的治理一致性"——文档可信度、接口传播、宪法咬合是持续关切，非事件驱动。若无常设委员会，以下问题无人问责：① 阶段关卡退出声明是否自洽（P-9）；② shared 接口变更的连锁影响是否被穷举（P-1 经验）；③ 设计-实现可追踪性是否断裂。Meso-Lite 已实证：常设机制空缺导致 17 条问题积累到反思阶段才被发现
- **触发条件**：已在 Meso-Lite 反思阶段触发——阶段关卡无人核验、READEME 错误无人阻止、设计契约无人持续盯
- **预估阶段**：已触发

### P-12：DocGovern agent（组织记忆载体）的宪法地位

- **宪法条款**：议题五 19 柱定义（四大皮层区 18 柱 + LoopController）——DocGovern 不在其中。宪法第七章（记忆系统）定义了个体记忆，未定义组织的结构化记忆载体
- **潜在违宪场景**：Meso-Lite 全程对话中发现了 8 条设计原则（P1-P8）、17 条文档错误、6 条工程实际约束——这些全部存在于聊天记录中，无结构化存储、无可检索索引、无关联到被修改文件。如果下一阶段启动时 DocGovern 不存在，本次反思的成果将全部流失，所有错误重犯。DocGovern 不是功能柱（它有独立于执行循环的职责），也不是常设委员会（它是记录者而非辩论者）——它是宪法未覆盖的一类全新架构角色
- **触发条件**：已在 Meso-Lite 反思阶段触发——整场反思的结论若无结构化载体，阶段跃迁时必然流失
- **预估阶段**：已触发。良性违宪——守了宪法精神（记忆不丢、治理可追溯、架构不自毁），违了宪法条文（19 柱之外的新角色）

### P-13：监理的宪法定义与治理角色冲突

- **宪法条款**：4.1.1——"质量与安全皮层区（搭档取向主激活，部分功能柱在监理取向下中激活）"。宪法定义监理为取向开关——切到监理取向，某些皮层区被激活，另一些被抑制。
- **治理概念讨论中的监理定义**：门下省——独立的封驳权，违宪即驳回，有自己的 LLM 调用通道（key-D），和 Meta-Agent 平级。
- **冲突**：一个开关 vs 一个司法分支——除了都叫"监理"，没有一毛钱关系。宪法未定义监理作为"独立于 Meta-Agent 的合规审查者"角色。如果治理层要引入监理封驳权，必须修宪——要么扩展 4.1.1 的监理定义，要么新增独立的监理实体（20 柱？）。
- **触发条件**：过渡阶段完成后、Core-2a（19 柱上线）前，治理层设计启动时
- **预估阶段**：Core-2a 前

### P-14：吏部/户部在 19 功能柱中的空缺

- **宪法条款**：4.1.1——19 功能柱定义（四大皮层区 + LoopController）。全部 19 柱均为执行柱：CodeGen、SecReview、Deploy 等。
- **治理概念讨论中的吏部/户部**：吏部管柱的激活/冻结/降权/重初始化。户部管 token 配额、key pool、上下文窗口预算。
- **冲突**：管人的和管钱的在 19 柱中没有对应物。不是功能柱（不执行具体项目任务），但又是必要的治理角色。要么扩展 19 柱定义（新增治理类柱），要么吏部/户部作为独立于 19 柱的治理分类层存在。
- **触发条件**：常设委员会启动后，出现"谁有权冻结某柱""谁分配 token 配额"的决策需求
- **预估阶段**：Core-2a 前

### P-15：冷启动 Meta-Agent 任命权空白

- **宪法条款**：4.1.2——Meta-Agent "是一个拥有调度职能的内部 Agent"。宪法定义了 Meta-Agent 的职责和约束，未定义谁来任命它、谁来罢免它。
- **治理概念讨论中的人事权**：阶段跃迁时常设委员会全体会议重确认 Meta-Agent。但冷启动时——第一个 Meta-Agent 从哪来？如果是宪法硬编码默认，那冷启动就是独裁起点，之后的人大/民主集中制度全部建在独裁地基上。
- **触发条件**：过渡阶段启动时，Meta-Agent 的初始实例化没有"任命来源"
- **预估阶段**：过渡阶段启动时（已触发）。过渡阶段仍沿用 Meso-Lite 的 Meta-Agent 实例作为延续——此空白在常设委员会就位后正式解决
