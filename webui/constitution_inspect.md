### 七条不可变原则速览

| 原则 | 核心内容 |
|------|---------|
| **原则一** | 确认永远在用户手里。L2/L3 不可逆操作必须经用户确认 |
| **原则二** | 规划与执行分离。MetaAgent 只规划不执行，Agent 只执行不规划 |
| **原则三** | 安全边界在 Toolkit 调用层。Agent 以身份调用，不持有权限定义 |
| **原则四** | 谁调用谁负责。Agent 对其工具调用后果承担全部责任 |
| **原则五** | 所有可观测事件走 PipelineObserver 统一管道 + 三档错误上报标准 |
| **原则六** | 用户是最终裁决者。多 Agent 并行产出须先经圆桌收束再呈用户 |
| **原则七** | 系统自我修改受宪法约束（v2.5.11 由 AM-2026-0515-001 新增） |

## 五、三轨定义分布

宪法中共有 **2 套三轨系统**，分布在两个不同维度：

### 5.1 错误上报三档（§8.1 SafeErrorReporter）

| 等级 | 语义 | 说明 |
|------|------|------|
| **fatal** | 致命错误 | 系统不可继续，通常触发 DECISION_REQUIRED |
| **degraded** | 降级错误 | 系统降级运行，触发 WARNING |
| **silent** | 静默错误 | 静默异常累积至阈值，升级为 WARNING |

> 来源：原则五 + §8.1 SafeErrorReporter——统一错误上报协议

### 5.2 通知三轨语义分层（§8.2 通知管线）

| 轨道 | 语义 | ButlerAgent 分发行为 |
|------|------|---------------------|
| **FYI** | 信息通知 | 静默记录→写入通知队列→闲时合并摘要呈现 |
| **WARNING** | 警告通知 | 状态灯变更 + 通知面板标记，不打断当前操作 |
| **DECISION_REQUIRED** | 需要决策 | 打断当前 UI→弹出决策界面→阻塞等待用户响应 |

> 来源：AM-2026-0515-002（v2.5.12）新增 §8.2

### 5.3 两套三轨的关系

- **错误三档**（§8.1）属于**错误维度**——关注"系统哪里出了问题"
- **通知三轨**（§8.2）属于**呈现维度**——关注"用户需要看到什么、何时看到、如何响应"
- 两者正交互补：fatal → DECISION_REQUIRED，degraded → WARNING，silent 计数器升级 → WARNING
